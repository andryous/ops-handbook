# Review and Approval

This section describes a simple review flow for operational content and task updates.

## Draft

The first version is created and saved for internal review.

## Review

A reviewer checks the content for clarity, accuracy, and completeness.

## Approval

Once confirmed, the content or task update is approved for use.

## Update control

Any later change should be reviewed again to keep the information reliable.
