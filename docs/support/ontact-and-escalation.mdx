# Contact and Escalation

This section explains who to contact when an issue cannot be resolved during normal support steps.

## First level

Start with the assigned team administrator or support contact.

## Second level

Escalate the issue when access, workflow, or system problems remain unresolved.

## Required information

Include the task ID, a short description of the issue, and any relevant screenshots or error messages.

## Priority

Critical issues should be reported as soon as possible to reduce operational impact.
